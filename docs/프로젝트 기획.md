# 프로젝트 기획: 온라인 간편 계산기

## 1. 프로젝트 기획

### 1.1. 프로젝트 목표
사용자들이 일상생활 및 특정 상황에서 자주 필요로 하는 다양한 계산 기능들을 한곳에 모아 제공하는 종합 유틸리티 웹 애플리케이션을 개발합니다. **Google AdSense**와 **Coupang Partners**를 연동하여 수익을 창출하는 것을 목표로 합니다.

### 1.2. 타겟 사용자
IT, 부동산, 금융, 학점, 게임 재화 등 특정 계산이 필요한 모든 인터넷 사용자.

### 1.3. 기술 스택
* **프론트엔드:** 순수 `HTML`, `CSS`, `JavaScript` (별도의 프레임워크 없이 구현)
* **CSS 프레임워크:** `Tailwind CSS` (CDN 방식)를 사용하여 빠르고 효율적인 반응형 UI 개발.
* **API 활용:** `CoinGecko` (암호화폐 시세), `ExchangeRate-API` (국가별 환율)
* **배포:** `Netlify`, `GitHub Pages` 등 정적 사이트 호스팅 활용.
* **개발 방식:** 모든 코드를 단일 `index.html` 파일에 통합하여 개발 및 배포의 복잡성을 최소화.

### 1.4. 주요 기능 및 수익화 전략

#### 제공되는 계산기 목록 (총 16종)
*   **IT/데이터:** 데이터 단위 (`TB`, `GB`, `MB`)
*   **금융/투자:** 주요 코인 환율, 세계 환율, 연봉 실수령액, 주식 수익률, 예/적금 이자, 대출 이자
*   **부동산:** 부동산 중개보수
*   **생활/건강:** 만 나이, BMI (비만도), D-Day, 퍼센트
*   **주소/위치:** 우편번호 찾기, 한/영 주소 변환기
*   **교육:** 학점
*   **게임:** 로벅스 환율

#### 수익화 모델
* **Google AdSense:** 사이트 내 디스플레이 광고 게재를 통한 광고 노출 및 클릭 수익 창출.
* **Coupang Partners:** 사이드바 및 콘텐츠 영역에 상품 추천 배너 삽입을 통한 제휴 마케팅 수익 창출.

## 2. MVP 개발 - 부동산 중개보수 계산기

### 2.1. 목표
검색 수요가 가장 높을 것으로 예상되는 '**부동산 중개보수 계산기**'를 최소 기능 제품(**MVP**)으로 우선 개발합니다. 이를 통해 프로젝트의 기본적인 구조를 확립하고 초기 시장성을 검증합니다.

### 2.2. 핵심 기능 정의
* **입력 항목:**
    * 거래 종류 (매매/교환, 임대차)
    * 부동산 종류 (주택, 오피스텔, 그 외)
    * 거래 가액 (매매가, 또는 보증금/월세)
* **출력 항목:**
    * 법정 상한 요율에 따라 계산된 '최대 중개보수' 금액.
    * 적용된 요율 및 한도액 정보.

### 2.3. 구현
단일 `HTML` 파일 내에 계산기의 기본 구조(`HTML`), 스타일(`CSS`), 로직(`JavaScript`)을 모두 구현합니다. 월세의 '**환산보증금**' 계산 로직을 포함하여 국토교통부 고시 최신 요율표에 기반한 정확한 계산 로직을 구현합니다.

## 3. 기능 확장 - 종합 계산기 포털로의 발전

### 3.1. 목표
다양한 주제의 계산기를 대거 추가하여 사이트의 활용도를 극대화하고, 더 넓은 사용자층을 유입시켜 종합 포털의 성격을 갖추는 것을 목표로 합니다.

### 3.2. 추가된 계산기 기능 명세

* **데이터 단위 계산기:**
    * **기능:** 테라바이트(`TB`), 기가바이트(`GB`), 메가바이트(`MB`) 단위를 상호 변환.
    * **특징:** 한 필드에 값을 입력하면 나머지 단위가 1024 단위를 기준으로 실시간 자동 계산되며, 세 자리 쉼표(`,`)를 적용하여 가독성을 확보.

* **코인/세계 환율 계산기:**
    * **기능:** 외부 API를 실시간으로 호출하여 코인 시세 및 국가별 환율 데이터를 가져와 상호 변환.
    * **특징:** 한 필드에 값을 입력하면 나머지 모든 필드의 값이 실시간으로 계산되는 동적인 UX 제공.

* **연봉 실수령액 계산기:**
    * **기능:** 연봉, 퇴직금 포함 여부, 부양가족 수를 기반으로 4대 보험 및 소득세를 공제한 월 실수령액 계산.
    * **특징:** **2025년 기준** 4대 보험 요율 및 근로소득 간이세액표를 적용하여 정확도 확보.

* **예/적금 이자 계산기:**
    * **기능:** 단리/복리, 과세/비과세 옵션을 선택하여 만기 시 세후 수령액을 정밀하게 계산.

* **로벅스 환율 계산기:**
    * **기능:** 게임 '로블록스'의 재화인 로벅스(Robux)를 원화(KRW), 달러(USD)와 상호 변환.
    * **특징:** PC/웹과 모바일의 다른 가격 정책을 반영하는 '구매 플랫폼' 선택 기능을 추가하여 실용성을 높임.

* **만 나이, D-Day, BMI 계산기:**
    * **기능:** 생년월일, 특정 날짜, 신체 정보를 입력받아 직관적인 결과(만 나이, 남은 날짜, 비만도 등)를 제공.

* **대출 이자 계산기:**
    * **기능:** 원리금 균등분할상환 방식을 기준으로, 대출 원금, 기간, 금리에 따른 월 상환금 및 총 이자 계산.

* **학점 계산기:**
    * **기능:** 사용자가 과목, 학점, 성적(A+, F 등)을 동적으로 추가/삭제하며 평균 학점을 계산.
    * **특징:** P/F 과목을 평점 계산에서 제외하는 옵션을 제공하여 편의성 증대.

## 4. UI/UX 개선 작업

### 4.1. 목표
다양해진 기능을 체계적으로 보여주고, 사용자 편의성을 극대화하기 위해 사이트 구조와 디자인을 전면적으로 개선합니다.

### 4.2. 주요 개선 사항

* **레이아웃 전면 개편:**
    * 단일 컬럼 구조에서 '좌측 고정 사이드바 + 중앙 콘텐츠' 형태의 2단 컬럼 레이아웃으로 변경.
    * 데스크톱 환경에서 내비게이션 접근성과 광고 배너 주목도를 크게 향상.

* **모바일 내비게이션 혁신 (`v1.1` 업데이트):**
    * **가로 스크롤 메뉴 도입:** 기존의 숨겨진 '햄버거 메뉴' 방식은 사용자가 다른 계산기의 존재를 인지하기 어렵다는 문제점을 개선하기 위해, 모바일 화면 상단에 모든 계산기 목록을 한눈에 볼 수 있는 가로 스크롤 메뉴를 도입.
    * **시인성 및 접근성 극대화:** 새로운 UI를 통해 사용자가 다른 기능들을 즉시 인지하고 쉽게 탐색할 수 있도록 시인성과 접근성을 획기적으로 개선.

* **반응형 광고 배치:** 쿠팡 배너를 데스크톱에서는 사이드바 하단에, 모바일에서는 콘텐츠 상단에 배치하여 기기별 레이아웃을 최적화.

* **사용자 경험(UX) 고도화:**
    * **아이콘 도입:** 데스크톱 사이드바 메뉴에 각 계산기를 상징하는 SVG 아이콘을 추가하여 시인성 강화.
    * **결과 복사 기능:** 모든 계산 결과에 '복사하기' 버튼을 추가하여 사용자가 결과를 쉽게 활용할 수 있도록 함.
    * **숫자 자동 포맷팅:** 모든 숫자 및 금액 입력 필드에 세 자리 쉼표(`,`)를 자동으로 삽입/제거하여 가독성과 일관성을 높임.
    * **탭(Tab) 기반 UI:** URL 해시(`#`)와 연동되는 탭 시스템을 구현하여, 특정 계산기로의 직접 링크 및 브라우저 히스토리 관리가 가능하게 함.

* **정보 시각화 및 가독성 개선:**
    * 각 계산기 하단에 관련 정보 및 팁을 제공하는 '**정보 섹션**'을 추가.
    * '**부동산 중개보수 요율표**'와 같이 복잡한 정보를 가독성 높은 카드형 디자인으로 시각화.

## 5. 배포 및 수익화

### 5.1. 목표
개발된 웹 애플리케이션을 실제 인터넷 환경에 배포하고, 수익 모델을 적용하여 운영을 시작합니다.

### 5.2. 배포 과정
* **버전 관리 및 자동화:** `GitHub` 저장소와 `Netlify`를 연동하여, 코드 Push 시 자동으로 사이트가 업데이트되는 `CI/CD`(지속적 통합/배포) 환경을 구축.
* **도메인:** `Netlify`에서 제공하는 무료 서브도메인(`netlify.app`)을 사용.

### 5.3. 수익화 연동
* **Google AdSense:** 사이트 심사 통과 후, `<head>` 태그에 애드센스 코드를 삽입하여 광고 시스템 활성화.
* **Coupang Partners:** 발급받은 동적 배너 스크립트를 데스크톱 및 모바일 레이아웃에 최적화된 위치에 각각 삽입.

## 6. 검색엔진 최적화 (SEO)

### 6.1. 목표
구글, 네이버 등 주요 검색엔진에서 사이트가 잘 노출되도록 하여, 자연적인 방문자(Organic Traffic) 유입을 극대화하는 것을 목표로 합니다.

### 6.2. On-Page SEO (사이트 내부 최적화)
`index.html` 파일의 `<head>` 태그 내에 12종의 모든 계산기를 포괄하는 메타(Meta) 태그를 전략적으로 구성합니다.

```html
<title>온라인 간편 계산기 | 데이터 단위, 코인 환율, 세계 환율, 만나이, 연봉, 부동산, 예적금, BMI, D-Day, 대출, 학점</title>

<meta name="description" content="데이터 단위(TB, GB, MB), 코인/국가 환율, 만 나이, 연봉 실수령액, 부동산 중개보수, 예적금 이자, BMI, D-Day, 대출 이자, 학점까지 모든 것을 간편하게 계산하세요.">

<meta name="keywords" content="데이터 단위 계산기, 온라인 계산기, 만능 계산기, 코인 계산기, 환율 계산기, 만나이 계산기, 연봉 계산기, 부동산 계산기, 이자 계산기, BMI 계산기, D-Day 계산기, 대출 이자 계산기, 학점 계산기">
```

### 6.3. Off-Page SEO (외부 최적화) 전략
* **검색엔진 등록:** 구글 서치 콘솔, 네이버 서치어드바이저에 사이트맵을 제출하여 검색엔진이 사이트 구조를 효율적으로 수집하도록 함.
* **콘텐츠의 지속적인 관리:** 법령이나 요율 변경(예: 4대 보험 요율) 시 계산 로직을 신속하게 업데이트하여 정보의 정확성과 신뢰성을 유지.
* **외부 링크 확보 (Backlink):** 관련 주제의 온라인 커뮤니티(IT, 부동산, 재테크, 게임 등)나 블로그에 유용한 정보로 사이트를 자연스럽게 소개해 양질의 외부 링크를 확보.

## 7. 프로젝트 개선 이력

### v1.1 (2025-09-02):
* **모바일 UI 개선:** 사용성 피드백을 반영하여, 기존의 숨겨진 햄버거 메뉴를 제거하고 모든 계산기 목록을 상단에서 바로 확인할 수 있는 가로 스크롤 메뉴로 교체. 이를 통해 모바일 사용자의 기능 탐색 편의성을 대폭 향상.
* **API 오류 처리 강화:** `CoinGecko`, `ExchangeRate-API` 호출 실패 시, 사용자에게 명확한 오류 메시지를 안내하고 관련 입력 필드를 비활성화하여 오작동을 방지하는 로직 추가.

### v1.2 (2025-09-07):
* **코인 계산기 기능 강화:**
    * **신규 코인 추가:** 스테이블 코인인 테더(USDT)와 USD 코인(USDC)을 추가하여 변환 기능 확장.
    * **시인성 개선:** 각 코인 이름 앞에 있던 제네릭 아이콘을 실제 코인 로고 이미지로 변경하여 사용자가 코인을 더 쉽게 식별할 수 있도록 개선.
* **모바일 내비게이션 편의성 향상:**
    * **스크롤 버튼 추가:** 모바일 화면 상단의 가로 스크롤 메뉴 양 끝에 좌/우 화살표 버튼을 추가.
    * **동적 버튼 표시:** 사용자의 스크롤 위치에 따라 버튼이 자동으로 나타나거나 사라지도록 구현하여, 스크롤이 가능하다는 점을 시각적으로 안내하고 조작 편의성을 높임.

### v1.3 (2025-09-12):
* **프로젝트 구조 개선:** 단일 `index.html` 파일에 통합되어 있던 `CSS`와 `JavaScript` 코드를 각각 `src/css/style.css` 및 `src/js/main.js` 파일로 분리하여 코드 관리 및 유지보수성을 향상.
* **보안 강화 (CSP):** `index.html`에 `Content Security Policy (CSP)` 메타 태그를 추가하여 외부 리소스 로딩을 제어하고 보안 취약점을 개선.
* **백엔드 로직 이전 (Netlify Functions):**
    * 주요 계산기 로직(만 나이, 연봉, 부동산, 이자, BMI, D-Day, 대출, 학점)을 Netlify 서버리스 함수로 이전하여 클라이언트로부터 핵심 로직을 분리.

### v1.4 (2025-09-13):
* **콘텐츠 보안 정책(CSP) 업데이트:**
    * Google AdSense 및 Coupang Partners 광고가 정상적으로 로드되도록 `index.html` 파일의 CSP `script-src` 및 `frame-src` 지시문을 업데이트하여 광고 관련 콘솔 오류를 해결.
* **백엔드 오류 수정 (암호화폐 계산기):**
    * **원인 분석:** 넷리파이 함수 로그 분석 결과, CoinGecko API가 API 키 없는 서버 요청을 차단하고 있음을 확인.
    * **조치:** 무료 Demo API 키를 발급받아 넷리파이 환경 변수에 등록하고, `calculate-crypto.js` 함수가 이 키를 사용하도록 수정하여 500 서버 오류 해결.

### v1.5 (2025-09-13):
* **전체 계산기 백엔드 이전 완료:**
    * **세계 환율 계산기:** `calculate-exchange-rate.js` 함수로 이전 완료.
    * **로벅스 계산기:** `calculate-robux.js` 함수로 이전 완료. 이로써 모든 계산기의 핵심 로직이 서버리스 백엔드로 이전되어 유지보수성 및 보안성이 향상됨.
* **Netlify 빌드 오류 해결:**
    * **원인 분석:** Netlify 빌드 과정에서 함수별 `package.json`의 의존성을 자동으로 설치하지 않아 `node-fetch` 모듈을 찾지 못하는 오류 발생.
    * **조치:** `netlify.toml` 파일에 `@netlify/plugin-functions-install-core` 플러그인을 추가하여, 빌드 시 함수 의존성을 자동으로 설치하도록 설정하여 문제 해결.
